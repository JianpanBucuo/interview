## 主流索引查找算法
- 线性查找 Linear Search
- 二分查找 Biary Search
- 二叉查找树 Binary tree Search
- 平衡二叉树 AVL tree
- B树 B Tree
- B+树 

https://www.cs.usfca.edu/~galles/visualization/Algorithms.html

### 线性查找
时间复杂度 O(n)

从第一个数据开始逐个匹配


### 二分查找 （有序）
 时间复杂度O(logN)
 拿出有序数列的中点位置作为比较对象
 根据中点数据大小，选取一半数据作为新的数列
 每次可以将数据量减少一半

 ### 二叉查找树

 有平衡的问题，如果一直往最右边加容易退化成`链表线性查找`

 时间复杂度 O(logN)
 不存在二分查找找不到中点的问题
 容易退化成链表O(n)

 ### 平衡二叉树
 会有左旋右旋的操作
 不会退化成线性查找
